Network Reconnaissance and Port Scanning Lab Report

Objective:

Discover open ports and services in the local network, analyze exposure, and understand network security risks using Nmap and Wireshark.

Nmap Scan Methodology

  • Tool: Nmap (nmap -sS 192.168.1.0/24)
  • Method: TCP SYN scan across all hosts within the network range.
  • Purpose: Identify devices, enumerate open, closed, and filtered ports.

Security Risk Analysis

  • FTP (21/tcp): Unencrypted, vulnerable. Disable or upgrade to SFTP.
  • HTTP (80/tcp): Traffic not encrypted; always use HTTPS.
  • SMB (445/tcp): Common attack vector; keep patched and restrict access.
  • UPnP (5000/tcp): Potentially unnecessary; disable if not required.
  • Kerberos (88/tcp): Secure protocol, but verify correct configuration.
  • Filtered SSH/Telnet: Indicates possible protection, further review for unwanted exposure.
  • Unknown Ports (5678, 7000): Investigate device type/usage.

Wireshark Analysis Steps

  • Install and start Wireshark, select main network interface.
  • Apply filter (example): ip.addr == 192.168.1.3
  • Observe SYN, TCP, TLS, and QUIC packets.
  • Capture packet types and service activity relevant to Nmap findings.

Key Concepts and Learning Outcomes

  • Port scanning reveals device exposure.
  • TCP SYN scan: quickly identifies open services stealthily.
  • Service enumeration and risk documentation.
  • Risk mitigation: Remove/secure unnecessary services, apply firewall controls.

Recommendations

  • Regular port scans to catch unauthorized services.
  • Use secure versions of protocols; disable legacy services.
  • Apply network segmentation and software updates.
